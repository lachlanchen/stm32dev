# ESP32-S3 TSL2591 Status, 2026-07-01

## Working Hardware

Controller:

```text
Waveshare ESP32-S3-DEV-KIT-N8R8
USB upload/serial port: COM6
CH343 UART port also appears as: COM5
```

Sensor:

```text
TSL2591 intensity sensor
I2C address: 0x29
```

## Working Wiring

```text
ESP32-S3 3V3 -> TSL2591 VCC / VIN
ESP32-S3 GND -> TSL2591 GND
ESP32-S3 IO9 -> TSL2591 SCL
ESP32-S3 IO8 -> TSL2591 SDA
TSL2591 INT   -> leave unconnected
```

Do not connect the sensor to STM32 and ESP32 at the same time.

## Firmware

Sketch:

```text
firmware/esp32_tsl2591_reader/esp32_tsl2591_reader.ino
```

Compile/upload command:

```powershell
$fqbn='esp32:esp32:esp32s3:USBMode=hwcdc,CDCOnBoot=cdc,PSRAM=disabled'
arduino-cli compile --fqbn $fqbn firmware\esp32_tsl2591_reader
arduino-cli upload -p COM6 --fqbn $fqbn firmware\esp32_tsl2591_reader
```

The firmware directly talks to TSL2591 over Wire/I2C without an external sensor library. It prints CSV:

```text
t_ms,present,configured,full,ir,visible
```

where:

```text
visible = full - ir
```
